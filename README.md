# site-web-lab

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库内包含若干由不同目的生成的静态网页文件，主要用于测试、展示或记录前端开发过程中的独立页面。

## 目录说明

```
site-web-lab/
├── pages/          # 存放各独立 HTML 页面及其相关资源
├── assets/         # 页面共享的样式、脚本、图片等静态资源
├── README.md       # 本文件
└── index.html      # 仓库首页（可选，用于页面导航）
```

- `pages/`：每个子文件夹代表一个独立页面项目，内含 `.html` 文件及该页面专属的 CSS、JS 等。
- `assets/`：存放多个页面共用的资源，如全局样式库、通用图标等。

## 页面归档说明

本仓库不针对任何特定域名或网站，每个 HTML 页面均为独立存在，可能包含：
- 前端技术演示
- 布局或组件原型
- 学习笔记对应的静态页面
- 临时测试页面

页面之间无强制依赖关系，可单独通过浏览器打开或部署至任意静态托管服务。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/site-web-lab.git
   ```
2. 直接在浏览器中打开 `pages/` 下的任意 `.html` 文件查看效果。
3. 如需整体浏览，可访问仓库根目录的 `index.html`（如存在）。

## 维护说明

- 新增页面时，请在 `pages/` 下创建以页面功能命名的文件夹，并放置相关文件。
- 若页面使用了共享资源，请确保在 `assets/` 中引用，避免重复存放。
- 请保持目录结构清晰，每个页面文件夹内建议附带简短的 `README.md` 说明页面的用途与依赖。
- 本仓库不提供自动部署配置，使用者可自行通过 GitHub Pages、Netlify 等平台发布。
